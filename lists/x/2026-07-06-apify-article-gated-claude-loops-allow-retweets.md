  Yes — but the honest answer is narrower and more interesting than the hype version.
  
  As of Fable-class frontier models, there are now credible examples of LLMs or LLM-powered agents producing genuinely new results: new mathematical constructions, improved algorithms, deployed systems optimizations, wet-lab protocol improvements, reaction-condition discoveries, and experimentally validated biological hypotheses. But the pattern is not “a chatbot thinks for a while and invents a new science.” The pattern is:
  
  LLM + search loop + verifier/test/lab/simulator + human audit = real discovery.
  
  That distinction matters. The most impressive examples are not raw language-model outputs. They are agentic discovery systems where the model proposes candidates and an external truth mechanism kills bad ideas.
  
  Anthropic’s Fable 5 framing is a useful marker because it emphasizes long-horizon work, vision, memory, and long-context task persistence; Anthropic’s Claude Science workbench also points toward audited scientific workflows with compute, tools, connectors, and reviewer agents. But the strongest public “discovery” examples so far are mostly from OpenAI, Google DeepMind, Google Research, and AI-for-science lab collaborations rather than a clean “Fable alone discovered X” story.
  
  The core answer
  
  Yes, it has happened.
  
  But not evenly across all domains.
  
  It has happened most convincingly in domains where novelty can be checked quickly:
  
  mathematics with proof checking or expert verification;
  algorithms with automatic evaluators;
  code optimization with benchmarks;
  biology and chemistry with high-throughput experiments;
  lab protocols where success can be measured empirically;
  constrained scientific hypotheses where experiments can validate or falsify the claim.
  
  It has happened much less convincingly in domains where “new idea” means an unverified theory, a speculative essay, a vague business concept, or an impressive-sounding explanation without a hard feedback loop.
  
  The useful framing is not:
  
  “Can LLMs make discoveries?”
  
  The better framing is:
  
  “Under what verification conditions do LLM agents become reliable novelty generators?”
  
  That is where the evidence is strongest.
  
  The best examples so far
  1. OpenAI’s unit-distance conjecture result — probably the strongest “real discovery” example
  
  This is the cleanest example if you want something close to “an LLM made a novel mathematical discovery.”
  
  OpenAI reported that an internal general-purpose reasoning model disproved a longstanding conjecture in the planar unit-distance problem, a problem originally posed by Erdős in 1946. OpenAI says the result was checked by external mathematicians and involved unexpected ideas from algebraic number theory.
  
  The companion arXiv paper is important because it clarifies the human/AI split: it describes a human-verified version of the OpenAI-generated counterexample, says the proof was “human-digested, simplified, and generalized,” and notes that the initial proof was generated by an internal OpenAI model before exposition/refinement through human interactions with Codex.
  
  Why this matters:
  
  This is not just “the model suggested a theorem.” It produced a construction/proof relevant to a real open problem. The result survived expert mathematical scrutiny. That puts it in a different class from most AI-generated “research ideas.”
  
  The caveat:
  
  The mathematical result appears to be real. The exact degree of autonomy is harder for outsiders to audit, because the full generation trace is not public. So the clean version is: an AI-generated result, later human-verified and human-refined, solved/disproved a meaningful longstanding conjectural direction.
  
  That is still a very big deal.
  
  The deeper lesson:
  
  This example suggests that frontier models may now be capable of doing something like latent bridge discovery: connecting remote mathematical areas that humans did not expect to apply. OpenAI’s own writeup emphasizes the surprising use of algebraic number theory in a geometry/combinatorics problem.
  
  A good line for your post:
  
  The strongest evidence is no longer toy benchmarks. It is AI-generated mathematics that survives expert verification.
  
  2. AlphaEvolve — LLM agents discovering better algorithms and deployed optimizations
  
  Google DeepMind’s AlphaEvolve is one of the most important examples because it shows the real discovery pattern clearly: an LLM proposes code, automated evaluators score it, and the system evolves better algorithms over many iterations.
  
  DeepMind describes AlphaEvolve as an evolutionary coding agent powered by Gemini models and automated evaluators. It has reportedly found optimizations used inside Google’s infrastructure, including data-center scheduling improvements recovering an average of 0.7% of worldwide compute resources, TPU circuit rewrites, a 23% speedup in a Gemini architecture kernel leading to a 1% training-time reduction, and up to 32.5% speedups for a FlashAttention kernel.
  
  The math/algorithmic side is even more interesting. AlphaEvolve found new matrix multiplication algorithms, including a 4×4 complex matrix multiplication method using 48 scalar multiplications; the paper describes this as the first improvement after 56 years over Strassen’s algorithm in that setting.
  
  It also improved results on some open mathematical/computational problems, including a new lower bound in the 11-dimensional kissing number problem. DeepMind says it rediscovered state-of-the-art solutions in roughly three-quarters of tested open problems and improved prior bests in about one-fifth.
  
  Why this matters:
  
  AlphaEvolve is not just “AI writes code.” It is an example of LLM-guided search over executable ideas. The model is valuable because it can mutate, recombine, and invent candidate programs; the evaluator is valuable because it prevents the system from rewarding persuasive nonsense.
  
  The caveat:
  
  This is not open-ended scientific reasoning in the human sense. It shines when the target can be written as code and scored automatically. But many important discoveries are exactly like that: algorithms, kernels, scheduling policies, numerical methods, search heuristics, compression methods, proof strategies, circuit layouts.
  
  The deeper lesson:
  
  The verifier is the microscope. The LLM is the hypothesis generator.
  
  Without the verifier, you get plausible text. With the verifier, you get selection pressure.
  
  3. FunSearch — new mathematical constructions through LLM-generated programs
  
  FunSearch, also from Google DeepMind, is an earlier but still very important example. It combines an LLM with a systematic evaluator, then searches over programs rather than over final text answers.
  
  In Nature, DeepMind reported that FunSearch surpassed best-known results in the cap set problem and found online bin-packing heuristics that beat standard baselines. The key architecture was a pretrained LLM paired with an evaluator that scores candidate programs, preventing confabulated answers from being accepted.
  
  Why this matters:
  
  FunSearch demonstrated a crucial inversion:
  
  Instead of asking the model to output the answer, ask it to output a program that can produce or verify the answer.
  
  That makes the output more inspectable. A program can be run, benchmarked, simplified, mutated, and studied. This is a much better discovery substrate than prose.
  
  The caveat:
  
  FunSearch is strongest in problems where the search target can be represented as code and evaluated. It is less helpful for domains where the “truth function” is slow, ambiguous, or expensive.
  
  The deeper lesson:
  
  FunSearch and AlphaEvolve together suggest that one of the most powerful roles for LLMs is not “oracle,” but creative mutation engine inside a Darwinian loop.
  
  A sharp way to say it:
  
  The breakthrough is not that LLMs know the answer. It is that they can generate unusually fertile candidate structures for evaluators to select from.
  
  4. OpenAI’s GPT-5 science acceleration examples — real but human-scaffolded
  
  OpenAI’s “early experiments in accelerating science with GPT-5” is not a single discovery but a collection of human-AI research collaborations across math, biology, physics, computer science, astronomy, and materials. OpenAI explicitly frames these as expert-guided collaborations, not autonomous science.
  
  The accompanying paper reports four new mathematical results verified by human authors. The authors are careful about scope: the results are described as modest, but the broader implication is that frontier models can contribute to real research progress when paired with experts.
  
  Examples OpenAI highlights include:
  
  a sharper step-size bound and proof in convex optimization;
  a mechanistic immunology insight involving N-linked glycosylation and memory T cells, followed by experiments that matched lab data;
  cleaning and extending work around an Erdős problem;
  graph theory inequalities produced with scaffolding;
  a stronger lower bound for an online algorithm.
  
  Why this matters:
  
  This is closer to the actual future of AI science than the cartoon version. The model does not replace the scientist. It expands the scientist’s search radius, generates mechanisms, proposes experiments, notices patterns, and helps close proof gaps.
  
  The caveat:
  
  OpenAI is explicit that the work is not a systematic benchmark of autonomous discovery. The writeup notes limitations including hallucinated citations, sensitivity to scaffolding, and the need for expert oversight.
  
  The deeper lesson:
  
  The first real AI discoveries may not look like “AI writes a Nobel paper.” They may look like “an expert team becomes 3×, 10×, or 100× more exploratory because the model can generate candidate mechanisms and proof paths on demand.”
  
  5. GPT-5 molecular cloning protocol improvement — a strong wet-lab example
  
  OpenAI and Red Queen Bio reported that GPT-5 improved a molecular cloning workflow by 79× in recovered sequence-verified clones. GPT-5 proposed, analyzed, and iterated protocol changes over multiple experimental rounds; scientists executed the protocol and uploaded the data. OpenAI says the system introduced a novel mechanism involving RecA and gp32.
  
  Why this matters:
  
  This is important because it is not just a simulation. It is a wet-lab optimization loop. The model proposed changes, the lab ran them, the measured result improved dramatically, and the loop continued.
  
  The caveat:
  
  This is not fully autonomous science. Humans still performed the wet-lab work and were part of the experimental loop. Also, the result is domain-specific: molecular cloning optimization, not a general-purpose invention engine.
  
  The deeper lesson:
  
  This is one of the most practically important forms of AI discovery:
  
  not inventing a new law of nature, but finding a protocol improvement no one had tried that dramatically changes yield, cost, or reliability.
  
  That kind of discovery can be economically huge even if it is not philosophically glamorous.
  
  6. GPT-5 + Ginkgo Bioworks — cheaper cell-free protein synthesis
  
  OpenAI and Ginkgo reported a closed-loop lab collaboration where GPT-5 designed cell-free protein synthesis experiments, the lab executed them, and data was fed back to the model. Across more than 36,000 unique CFPS compositions and six design-build-test cycles, they reported a new low-cost state of the art with a 40% lower cost than the previous baseline and a 57% improvement in reagent cost.
  
  Why this matters:
  
  This is a clean example of AI-guided experimental search. The model was not just writing explanations. It was choosing experimental compositions under constraints, receiving empirical results, and improving the next batch.
  
  The caveat:
  
  OpenAI notes the limitations: one protein, one CFPS system, high-throughput setting, and human oversight. Generalization still needs to be shown.
  
  The deeper lesson:
  
  Biology may be one of the biggest near-term domains for LLM-powered discovery because the search spaces are gigantic, but high-throughput experiments can provide feedback. The model’s job is not to be omniscient. It is to make better bets.
  
  A strong formulation:
  
  In biology, the LLM does not need to know the full causal graph. It needs to choose experiments that collapse uncertainty faster than human intuition alone.
  
  7. GPT-5.4 + https://t.co/8ka0Dx6x2O — AI chemist finding useful reaction additives
  
  OpenAI and https://t.co/8ka0Dx6x2O reported a near-autonomous chemistry workflow using GPT-5.4 with https://t.co/8ka0Dx6x2O’s Maria platform. The system generated proposals, designed experiments, ran follow-ups, and identified additives such as TEMPO and related oxidants that improved Chan–Lam coupling yields across many tested substrates. OpenAI reports yield improvements for 88% of tested boronic acids and 83% of tested sulfonamides, with bench-scale repeats showing higher yields in 11 of 14 pairs.
  
  Why this matters:
  
  This is a very concrete discovery pattern: the model proposed a surprising intervention, the autonomous lab tested it at scale, and humans later validated the effect on the bench.
  
  The caveat:
  
  OpenAI is clear that this was near-autonomous, not fully autonomous. Human chemists still made important choices, and independent reproduction is still needed.
  
  The deeper lesson:
  
  This kind of discovery is not about elegant theory. It is about finding useful knobs in a large experimental space. That is exactly where LLM agents may become extremely valuable.
  
  A good way to say it:
  
  The first AI chemist may not look like a genius theorist. It may look like a tireless experimental strategist that proposes the weird additive combination a human would not prioritize.
  
  8. Google’s AI co-scientist — strong hypothesis engine, weaker as standalone discovery proof
  
  Google’s AI co-scientist is a multi-agent Gemini-based system for generating, critiquing, refining, and ranking scientific hypotheses. The Nature paper describes agents that generate, debate, evolve, and refine research hypotheses for experimental verification.
  
  Why this matters:
  
  This is close to what many people imagined when they predicted “AI scientists”: multiple agents generating hypotheses, critiquing each other, and using test-time compute to improve ideas.
  
  The caveat:
  
  Hypothesis generation is not the same as discovery. The key question is whether the hypotheses survive experiments and prior-art review. Co-scientist-style systems are promising, but they need hard validation to count as more than advanced ideation.
  
  The deeper lesson:
  
  Co-scientist is less important as “proof that AI discovered X” and more important as a template for future discovery systems:
  
  generation agent, critic agent, ranking agent, novelty checker, experiment planner, literature verifier, and human-in-the-loop execution.
  
  That architecture is probably close to the future.
  
  9. Google/Yale C2S-Scale — model-guided biological hypothesis with lab confirmation
  
  Google and Yale reported C2S-Scale, a 27B-parameter single-cell foundation model built on Gemma, which identified a drug combination that may make “cold” tumors more visible to the immune system. Lab tests reportedly confirmed that silmitasertib plus low-dose interferon amplified antigen presentation in the relevant setting.
  
  Why this matters:
  
  This is not exactly an LLM agent in the same sense as Codex or AlphaEvolve. It is a domain-specific biological foundation model. But it belongs in the discussion because it shows the broader pattern: large generative models can surface non-obvious biological interventions that are later experimentally tested.
  
  The caveat:
  
  It is a narrower biological model, not a general-purpose chat agent. Also, “potential therapy pathway” is not the same as a clinically proven therapy.
  
  The deeper lesson:
  
  The most valuable future systems may not be a single general model. They may be frontier LLMs orchestrating specialized scientific foundation models: protein models, cell models, chemistry models, simulators, symbolic solvers, and lab robots.
  
  10. Sakana’s AI Scientist — fascinating, but not the strongest evidence of deep discovery
  
  Sakana’s AI Scientist line of work is worth mentioning because it pushes toward end-to-end automated research: generating ideas, checking novelty, running experiments, writing papers, and reviewing results. A Nature article describes the system as using foundation models in a complex agentic setup to generate research ideas, execute experiments, write papers, and evaluate outputs.
  
  Why this matters:
  
  This is one of the clearest attempts to automate the entire research-paper loop.
  
  The caveat:
  
  It is not yet the best example of “truly new discovery.” Independent analysis has found problems such as coding failures, flawed or misleading results, hallucinated details, and work that can resemble rushed low-quality research.
  
  The deeper lesson:
  
  End-to-end autonomy is seductive but dangerous. It can produce an impressive-looking paper-shaped object before it produces reliable knowledge.
  
  A sharp line:
  
  Automating the form of science is easier than automating the epistemic discipline of science.
  
  This is why AI Scientist-style systems need strong external verifiers, replication, and adversarial review before they should be treated as discovery engines.
  
  11. Coscientist and ChemCrow — important lab-agent precursors, but mostly “agentic execution”
  
  Coscientist and ChemCrow are relevant as earlier LLM-powered chemistry-agent systems. Coscientist was described as an AI system that could design, plan, and perform complex experiments using LLMs plus tools such as internet search, documentation search, code execution, and laboratory automation. ChemCrow similarly connected an LLM to chemistry tools for synthesis, drug discovery, and materials tasks.
  
  Why this matters:
  
  These systems showed that LLMs could move beyond text and into tool-mediated laboratory planning.
  
  The caveat:
  
  They are better understood as agentic execution and planning systems than as clear examples of novel discovery. A system autonomously reproducing a known reaction is impressive automation, but it is not the same as discovering a new reaction, theorem, or mechanism.
  
  The deeper lesson:
  
  These systems created the bridge from “LLM as adviser” to “LLM as lab operator.” The next step is “LLM as lab strategist,” where it proposes experiments that generate new knowledge faster than human-led search.

  What has not happened yet
  
  The exaggerated prediction was something like:
  “LLMs will autonomously produce major new scientific theories and inventions across domains.”
  
  That has not really happened in the broad, repeatable, public, independently auditable way people imagined.
  
  What has happened is more specific:
  LLM agents can now generate candidate ideas, programs, proofs, protocols, and experiments that sometimes become genuinely new results when embedded inside a strong verification loop.
  
  The missing pieces are:
  Independent replication.
  Many of the best examples come from company labs or company partnerships. They may be real, but the discovery process is not always fully reproducible by outsiders.
  
  Provenance transparency.
  We often do not get the full prompt logs, candidate counts, failed attempts, human interventions, model versions, or prior-art search process.
  Autonomy accounting.
  “AI discovered X” can hide a lot: human problem selection, human scaffolding, human filtering, human experimental execution, human proof repair, human interpretation.
  
  Negative results.
  We mostly hear about the wins. We rarely see the denominator: how many attempts failed, how many plausible ideas were wrong, how many discoveries were rediscoveries.
  
  Generalization.
  A model that optimizes one wet-lab protocol or one algorithmic class does not automatically become a general inventor.
  
  Novelty auditing.
  A result can be new to the model operator but already known in an obscure paper, codebase, patent, thesis, or internal lab notebook.
  
  Impact maturity.
  A new construction, yield improvement, or algorithmic speedup is real. But it may not yet be field-changing.
  
  The non-sycophantic answer is:
  
  The “LLMs will make discoveries” prediction is no longer false. But it is also not fully fulfilled. We have early real discoveries in verifier-rich domains, not general autonomous science.
  
  The most important hidden distinction
  
  There are three different things people collapse into one phrase:
  1. Idea generation
  The model proposes something interesting.
  Example: “Maybe this biological pathway matters.”
  This is weak unless tested. Models are very good at plausible idea generation, but plausible ideas are cheap.
  2. Search-guided discovery
  The model proposes many candidates, and an external evaluator selects the winners.
  Example: AlphaEvolve or FunSearch.
  This is much stronger because the system has a hard feedback signal.
  3. Epistemic discovery
  The system produces a new result that enters the body of human knowledge.
  Example: a verified theorem, a deployed algorithmic improvement, a reproducible wet-lab result.
  This is the level that matters.
  Most online discussion confuses level 1 with level 3.
  
  A strong version of your question should explicitly ask for level 3.
  
  The “genius-level” framework I would add
  
  Use this as the standard for judging AI discovery claims.
  
  The AI discovery ladder
  
  Level 0 — Interesting suggestion
  The model says something plausible. No validation.
  
  Level 1 — Novel hypothesis
  The idea appears non-obvious and survives a basic prior-art search.
  
  Level 2 — Automated verification
  The idea passes a test, benchmark, simulator, formal checker, or computational evaluator.
  
  Level 3 — Human expert verification
  Domain experts check the proof, code, experiment, or reasoning.
  
  Level 4 — Independent replication
  Another lab or team reproduces the result.
  
  Level 5 — Practical deployment or field uptake
  The result is used in production, cited, built upon, or changes expert practice.
  
  Level 6 — Field-shaping discovery
  The result opens a new research direction, resolves a major problem, or creates a new technology class.
  
  Most current LLM discoveries are Level 2–4. A few may be Level 5. We do not yet have many unambiguous Level 6 cases.
  
  That is still progress.
  
  Obscure but useful thought inputs
  
  The invention is often the harness, not the model
  
  The real scientific object is not “GPT thought of X.” It is the whole machine:
  
  model + tools + memory + search + evaluator + lab + human judgment + audit trail.
  
  This is why “which model discovered it?” is sometimes the wrong question. The better question is:
  
  What epistemic loop turned model-generated candidates into trustworthy knowledge?
  
  Verifiers are the new peer reviewers
  
  In classical science, peer review comes late. In LLM discovery systems, the verifier comes early and constantly.
  
  The verifier might be:
  a compiler;
  
  a unit test;
  a theorem prover;
  
  a benchmark;
  a simulator;
  
  a wet-lab assay;
  a sequencing result;
  
  an expert review;
  a prior-art search;
  
  a replication lab.
  The stronger the verifier, the more seriously we should take the discovery.
  
  Search is not “less real” than insight
  People sometimes dismiss AlphaEvolve/FunSearch-style results as “just search.”
  
  That is too shallow.

  How to build a 1-person AI company that:
  
  - Runs locally
  - 100% open-source
  - No human employees, all agents
  - Real-time collaboration via email
  
  Multi-agent orchestration is not new. Plenty of frameworks already let agents hand off tasks, run in parallel, and talk to each other.
  
  So the interesting question is not whether agents can collaborate. It is what structure you use to make them collaborate.
  
  The common approach is to wire a graph of nodes and edges and reason about the plumbing yourself. It works, but you are learning a new abstraction just to describe who does what.
  
  There is a coordination structure we have trusted for a hundred years already: an organization.
  
  Every company runs the same way. People have roles, roles have reporting lines, and work moves up and down that chart without anyone relaying each message by hand.
  
  Map that onto agents and the whole thing gets intuitive. You lay out an org chart, each agent fills one role, you talk to the person at the top, and the org sorts out the work between them.
  
  You already know how a company works, so you already know how to run one here. There is no new abstraction to learn.
  
  That is exactly what Alook does. Each agent is a live Claude Code or OpenCode session with a defined role, a reporting line, and its own email inbox.
  
  The agents coordinate over email, the same way a team would. And it all runs locally through a runtime on your own machine, so nothing leaves your setup.
  
  You bring your own agent too. Claude Code and Codex both work, and if you would rather stay fully open source and local, OpenCode works the same way.
